# API 配置说明

这份文档只说明两件事：

- 钱包地址怎么添加
- `Binance` 和 `OKX` 的只读 API 怎么申请、怎么填进程序

## 先说结论

- 链上钱包不需要 API Key
- 交易所账户需要只读 API
- 不要开启交易权限
- 不要开启提币权限
- 导出文件不会包含 API Key、API Secret、Passphrase
- 导入配置后，交易所密钥需要重新填写

## 钱包怎么配置

### 支持范围

当前版本支持这三类地址：

- `EVM`：可按所选链查询原生币与 ERC-20
- `Solana`：查询原生 `SOL` 与 `SPL Token`
- `Bitcoin`：查询地址余额

### 添加步骤

1. 打开程序
2. 进入 `钱包`
3. 点 `添加钱包`
4. 选择链类型
5. 填入地址
6. 可选填写备注名
7. 提交

### 注意

- `EVM` 地址可勾选多条链，同一地址会分别查询每条链上的原生币与 ERC-20
- `Solana` 地址会返回原生 `SOL` 与可识别的 `SPL Token`
- 部分 `SPL Token` 或长尾资产可能没有实时价格，页面会显示为旧价或待补充
- `Bitcoin` 目前按地址余额查询

## Binance API 配置

### 你需要准备什么

- `API Key`
- `API Secret`

### 创建步骤

1. 登录 `Binance`
2. 进入 `API Management` 或 API 管理页面
3. 创建新的 API
4. 完成账户安全验证
5. 在权限设置里只保留 `Read` 或只读权限
6. 关闭交易权限
7. 关闭提币权限
8. 保存配置

### 程序里怎么填

1. 打开程序
2. 进入 `交易所`
3. 点 `绑定账户`
4. 选择 `Binance`
5. 填写备注名
6. 填入 `API Key`
7. 填入 `API Secret`
8. 提交

### 建议

- 优先创建专门给这个项目使用的只读 API
- 如果你长期使用，建议额外加 IP 白名单
- 如果加了 IP 白名单，记得把当前实际出口 IP 加进去

## OKX API 配置

### 你需要准备什么

- `API Key`
- `API Secret`
- `Passphrase`

### 创建步骤

1. 登录 `OKX`
2. 进入 API 管理页面
3. 创建新的 API
4. 设置 API 名称
5. 自定义一个 `Passphrase`
6. 权限选择只读
7. 不要开启交易
8. 不要开启提币
9. 保存配置

### 程序里怎么填

1. 打开程序
2. 进入 `交易所`
3. 点 `绑定账户`
4. 选择 `OKX`
5. 填写备注名
6. 填入 `API Key`
7. 填入 `API Secret`
8. 填入 `Passphrase`
9. 提交

### 注意

- `Passphrase` 不是登录密码
- 它是你创建 API 时自己设置的那一项
- 少了它，`OKX` 无法正常查询

## DeFi 数据源配置

当前版本的 DeFi 统计使用服务端数据源查询钱包在协议中的仓位。

### 当前实现

- 默认优先使用 `Mobula`
- 当 `Mobula` 对某条链未识别到仓位时，会尝试使用 `DeBank` 公共页面做兜底补全
- 仅查询 `EVM` 与 `Solana` 钱包
- `Bitcoin` 暂不支持 `DeFi` 仓位查询
- `DeFi` 模块默认低频刷新，避免过快消耗免费额度
- `DeFi` 统计暂不并入顶部总资产，避免与钱包余额重复计算

### 需要准备什么

在项目运行环境中配置：

```bash
MOBULA_API_KEY=your_mobula_key
```

### 注意

- 这个 key 只在服务端 Route Handler 中使用，不会暴露到浏览器
- 导出 JSON 不会包含任何 DeFi 数据源密钥
- 未配置 key 时，不会影响钱包和交易所主功能，只会让 DeFi 模块显示不可用
- 当前实现默认**不会**在未配置 `MOBULA_API_KEY` 时自动调用 demo API，避免你误以为已经在请求正式接口但额度没有变化
- 对于 `Mobula` 未覆盖的地址/协议，当前还会尝试抓取 `DeBank` 公共页面作为非官方兜底来源，因此展示结果可能是“近实时公共页面快照”而不是结构化链上仓位明细
- 如果你确实想在本地临时试用 demo API，需要显式设置：

```bash
MOBULA_USE_DEMO_API=true
```

## 程序里的保存方式

- 钱包地址、账户标签、设置、最近资产快照与总资产历史看板，保存在当前浏览器本地
- `API Key`、`API Secret`、`Passphrase` 只保留在当前会话，不会写入长期本地存储
- 导出 JSON 时，不会导出这些密钥，也不会导出资产快照与历史看板数据
- 导入 JSON 后，交易所账户会恢复结构和标签，但密钥要重新填
- 导出和导入都不会包含资产快照，导入后需要重新刷新一次

## 常见问题

### 1. 为什么钱包能加，但资产不完整

`EVM` 会分别查询所选链，但某条链节点超时、代币价格缺失、或长尾 ERC-20 未被公共数据源覆盖时，资产可能只返回部分结果。`Solana` 已支持原生 `SOL` 与 `SPL Token`，但部分代币可能没有价格。

### 2. 为什么交易所绑定后显示异常

常见原因有这些：

- `API Key` 填错
- `API Secret` 填错
- `OKX Passphrase` 填错
- API 没开只读权限
- API 被平台风控限制
- IP 白名单没配对

### 3. 为什么导入后交易所不能直接刷新

因为导入不会恢复密钥。出于安全考虑，密钥需要重新填写。

### 4. 为什么不建议开交易权限

这个项目只需要查询余额。开交易或提币权限没有必要，也会提高风险。

## 推荐的配置顺序

1. 先添加一个钱包地址，确认页面能正常刷新
2. 再去创建 `Binance` 或 `OKX` 的只读 API
3. 回到 `交易所` 页面绑定账户
4. 最后检查首页总览、资产明细和异常提示
